# Naming Analysis — wv6

## Critical (file↔name mismatch, misleading intent)

**`app/projects/[slug]/page.tsx` — `PostPage`**
Route is `/projects/[slug]`, not posts. → `ProjectPage`.

**`app/projects/[slug]/view.tsx` — `ReportView`**
Renders `null`; fires a pageview POST. Name describes nothing. → `TrackPageView` or `PageViewBeacon`.

**`app/projects/[slug]/header.tsx` — `Header`**
Collides with generic layout `Header`. → `ProjectHeader`.

**`app/api/incr/route.ts` — route segment `incr`**
Non-grep-friendly abbreviation; public URL. → `increment` or `views`. Inside: `buf` → `hashBuffer`; `isNew` → `firstViewForIp` (the actual semantic of `nx`).

## High (ambiguous API-shaped names)

**`components/Page.tsx` — prop `gutter`**
`<aside>` sidebar content, not spacing. "Gutter" means whitespace in typography. → `sidebar` or `aside`.

**`app/projects/_data.ts`**
- `ProjectMeta` vs `Project` — asymmetric. → `ProjectFrontmatter` (what MDX exports) + `Project` (hydrated).
- `mod` → `mdxModule`
- `ascending` → `projectsByDateAsc` (name the content, not the order)
- `idBySlug` → `slugToId` (match `keyToValue` convention)

## Medium (boolean + collection conventions)

**`components/NavBar.tsx`**
- `nav` array → `navItems` (plural, specific)
- `item.link` → `item.href` (matches anchor semantics)
- `active` → `isActive`

**`app/page.tsx`**
- `socials.map((s) => …)` → `(social)`; array stays fine
- `interests` locally OK, but the section header is "Currently" — pick one term

**`app/projects/page.tsx`**
- `.filter((p) =>` / `.map((p) =>` → `(project)`
- `sorted` → `sortedProjects` (or drop the intermediate)

## Minor (polish)

- `iso` param in `header.tsx` → `isoDate`
- `views` in `[slug]/page.tsx` → `viewCount`

## Themes

1. **Mechanism-over-intent** in side-effect components (`ReportView`) and generic layout names (`Header`, `Page`).
2. **Abbreviations on public surfaces** (`incr`, `buf`, `mod`, `iso`) — cheapest wins, no semantic risk.
3. **Single-letter locals in domain code** (`p`, `s`) — loop counters fine, domain objects not.
4. **Boolean `is`/`has` convention** applied inconsistently.
