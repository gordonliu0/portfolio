Analysis complete. Here are naming issues across the codebase, ordered by severity.

## Critical

### 1. `components/Page.tsx` — `Page`
Collides semantically with Next.js `page.tsx` convention, which denotes routable pages. This component is actually a **two-column layout with a sticky side gutter**. The name `Page` says nothing about what it does for callers.

**Suggest:** `ArticleLayout`, `GutterLayout`, or `SideNoteLayout`. The prop `gutter` also describes position, not purpose — consider `aside`/`sidebar`.

### 2. `app/projects/[slug]/page.tsx:17` — `PostPage`
This route renders a **project**, not a post. Inconsistent with every surrounding token (`/projects`, `getProject`, `Project`, `content/projects/`). Leftover from a blog scaffold.

**Suggest:** `ProjectPage`.

### 3. `app/projects/[slug]/view.tsx` — `ReportView` + file `view.tsx`
`ReportView` sounds like it *displays* a report. It actually **POSTs to increment the pageview counter** on mount and renders nothing. File name `view.tsx` is equally misleading (sounds like a presentational component).

**Suggest:** component `TrackPageView` or `PageViewBeacon`; file `track-view.tsx`.

### 4. `app/api/incr/route.ts` — route `/api/incr`
Names the mechanism (`incr` = Redis INCR) instead of the resource. Not greppable; doesn't convey intent.

**Suggest:** `/api/pageviews` (POST) or `/api/projects/[slug]/views`.

## High

### 5. `app/projects/[slug]/header.tsx:24` — `Header`
Generic. It's a project-specific header with a metadata `<dl>`, not a reusable site header (that role belongs to `NavBar`). Also confusable with the `<header>` HTML element.

**Suggest:** `ProjectHeader`.

### 6. `app/projects/[slug]/header.tsx:3` — `Props`
`Props` is unsearchable and collides with every other `Props` in the codebase. CLAUDE.md explicitly flags ungreppable names.

**Suggest:** `ProjectHeaderProps`.

### 7. `app/projects/_data.ts` — file `_data.ts`
`_data` is generic and describes a *category* of file. Underscore-prefix hides it from Next.js routing — that's fine — but the name itself should say what it loads.

**Suggest:** `projects.ts` or `load-projects.ts`.

### 8. `components/NavBar.tsx:6` — `nav` + `item.link`
Variable `nav` shadows the `<nav>` JSX element used below. Property `link` is non-idiomatic — HTML/React convention is `href`.

**Suggest:** rename array to `navItems`; rename property `link` → `href`.

### 9. `app/page.tsx:3` — `socials` items `{ label, value, href }`
`value` is vague. It's actually the display handle (`gordonliu0`, `gg1iu`).

**Suggest:** `handle` or `display`.

### 10. `app/page.tsx:9` vs UI label — `interests` / "Currently"
Section heading reads "Currently" but data is `interests`. Mismatch: either rename variable (`currentFocus`, `nowWorkingOn`) or rename the UI label ("Interests").

## Medium

### 11. `app/projects/_data.ts:36` — `ascending`
Names the *direction* of the sort, not the value. Reads confusingly at the callsite (`[...metas].sort(...)` already implies a sort).

**Suggest:** `byDateAscending`, or inline into the `Map` construction.

### 12. `app/projects/_data.ts:5` — `ProjectMeta` vs `Project`
`Project = ProjectMeta & { slug, id }` is backwards from how readers think: the richer type should be the plain noun. `ProjectMeta` is really the MDX frontmatter.

**Suggest:** rename `ProjectMeta` → `ProjectFrontmatter`, keep `Project` as-is.

### 13. `app/api/incr/route.ts:35` — Redis key prefix `"deduplicate"`
Other keys use noun namespaces (`pageviews:projects:<slug>`). `deduplicate` is a verb and doesn't fit the scheme.

**Suggest:** `dedupe:<hash>:<slug>` or `pageviews:dedupe:<hash>:<slug>`.

### 14. `app/projects/_data.ts:21` — `getAllSlugs`
Doesn't filter unpublished, doesn't sort — just a filesystem read. `getAll*` suggests the domain list.

**Suggest:** `listProjectSlugFiles` or `readProjectSlugs`.

### 15. `components/NavBar.tsx:25` — `active`
CLAUDE.md's naming section calls out `is`/`has` prefix for booleans.

**Suggest:** `isActive`.

---

## Minor

### 16. `app/projects/_data.ts:32` — `mod`
Short scope, acceptable, but `projectModule` reads better given the `.mdx` dynamic-import pattern repeats in `getProject` too.

### 17. `app/api/incr/route.ts:26` — `buf`
Fine in scope. `digest` matches the crypto API it's the result of.

### 18. `app/page.tsx:65` — `s` in `.map((s) =>` 
One-letter name for a structured object in multi-line JSX. CLAUDE.md says scale specificity to scope — here a few lines of JSX reference it, so `social` reads better.

### 19. `app/projects/_data.ts:5` — `published?: boolean` with `!== false` check
Tri-state boolean (undefined = published). Either rename to `draft?: boolean` (inverted, default-falsy reads cleanly) or make `published` required with a default.
